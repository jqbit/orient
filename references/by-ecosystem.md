# Yorient by Ecosystem

Fast cheatsheet for the broad scan in `## Workflow §2`. For each ecosystem, lists the manifest files that matter, the conventional source/test/command layout, and what "read first" usually means. Use this to seed an explorer's exploration, not to substitute for actually looking.

## Node / TypeScript

**Manifests:** `package.json`, `tsconfig.json`, `.nvmrc` or `package.json#engines`, `.npmrc`.
**Lockfiles:** `package-lock.json` (npm), `pnpm-lock.yaml`, `yarn.lock`, `bun.lockb`.
**Monorepo flags:** `pnpm-workspace.yaml`, `package.json#workspaces`, `turbo.json`, `nx.json`, `lerna.json`, `rush.json`.
**Source layout:** `src/`, `lib/`, `apps/`, `packages/`, `services/`.
**Test layout:** colocated `*.test.ts` / `*.spec.ts`, or top-level `tests/` / `__tests__/`.
**Read first:** root `package.json` scripts, then workspace package manifests.
**Commands live in:** `package.json#scripts`; monorepo task graph in `turbo.json` or `nx.json`.
**Common no-go:** `node_modules/`, `dist/`, `build/`, `.next/`, `.turbo/`, `coverage/`, lockfiles (regenerated).
**Frameworks worth flagging in YORIENT.md:** Next.js (`app/` vs `pages/`), Remix, Nest (modules), Expo (`apps/*/ios|android/`), Vite, Astro.

## Python

**Manifests:** `pyproject.toml`, `setup.cfg`, `setup.py` (legacy), `requirements*.txt`, `Pipfile`, `environment.yml`.
**Lockfiles:** `poetry.lock`, `uv.lock`, `Pipfile.lock`, `pdm.lock`.
**Version pinning:** `.python-version`, `pyproject.toml#project.requires-python`.
**Source layout:** `src/<pkg>/`, or flat `<pkg>/` at root.
**Test layout:** `tests/` at root (pytest convention); `tests/` mirroring `src/` is common.
**Read first:** `pyproject.toml` (`[project]`, `[tool.*]` sections — esp. `[tool.poetry]`, `[tool.uv]`, `[tool.ruff]`, `[tool.pytest.ini_options]`).
**Commands live in:** `pyproject.toml#tool.*.scripts` (poetry / uv / pdm), or `Makefile`, or `tox.ini`.
**Common no-go:** `.venv/`, `venv/`, `__pycache__/`, `*.egg-info/`, `dist/`, `build/`, generated `_pb2.py` / OpenAPI stubs.
**Frameworks worth flagging:** Django (`settings.py`, `urls.py`, `apps/`), FastAPI (`main.py` or `app.py`), Flask (`app.py`), pytest plugins (`conftest.py`).

## Rust

**Manifests:** `Cargo.toml` (workspace at root, member crates under `[workspace.members]`), `rust-toolchain.toml`, `.cargo/config.toml`.
**Lockfiles:** `Cargo.lock` (committed for binaries; optional for libraries).
**Source layout:** `src/main.rs` or `src/lib.rs`; `src/bin/` for additional binaries.
**Test layout:** unit tests colocated under `#[cfg(test)] mod tests`; integration tests in `tests/`; benches in `benches/`; examples in `examples/`.
**Read first:** root `Cargo.toml` workspace section, then each member crate's `Cargo.toml`.
**Commands live in:** Cargo subcommands (`cargo build`, `cargo test`, `cargo clippy`, `cargo fmt`); rarely a Makefile or `just` recipe.
**Common no-go:** `target/`, `Cargo.lock` regeneration for libraries.
**Frameworks worth flagging:** Axum / Actix routes, Bevy ECS structure, Tauri (`src-tauri/`), embassy / esp-rs targets.

## Go

**Manifests:** `go.mod`, `go.sum`; workspace mode via `go.work` + `go.work.sum`.
**Source layout:** Standard Go project conventions — `cmd/<binary>/main.go`, `pkg/` (public), `internal/` (private), `api/`, `web/`.
**Test layout:** `_test.go` files colocated with sources; integration tests under build tags (`//go:build integration`).
**Read first:** `go.mod` (module path + Go version), then `cmd/*/main.go` for each binary.
**Commands live in:** Makefile, `mage`/`task` targets, or raw `go` commands documented in README.
**Common no-go:** `vendor/` (if present, generated by `go mod vendor`), build output dirs, `*.pb.go` (protobuf-generated).
**Frameworks worth flagging:** Gin / Echo / Chi routers, sqlc generated code, Wire dependency injection, Buf for protobuf.

## Multi-language / monorepo flags

When a repo mixes ecosystems, look at the build orchestrator before the per-language manifests:

- `turbo.json` — task graph across packages.
- `nx.json` + `workspace.json` — Nx workspace.
- `pnpm-workspace.yaml` + root `package.json#workspaces` — pnpm/npm workspace.
- `Cargo.toml#workspace` — Rust workspace.
- `go.work` — Go workspace.
- `Bazel/Buck` (`BUILD`, `WORKSPACE`, `MODULE.bazel`) — language-agnostic build.
- `BUCK`, `BUILD.bazel` — same, indicates a polyglot build.
- `.bazelrc`, `tools/` — repo-wide tooling.
- `Earthfile` (Earthly), `Dockerfile.dev` — sealed build environments.

For polyglot repos, list each ecosystem's entrypoint section in `YORIENT.md` rather than burying everything under one `## Commands`.

## Docs sites

- **mdBook:** `book.toml`, `src/SUMMARY.md`.
- **Docusaurus:** `docusaurus.config.js`, `docs/`, `sidebars.js`.
- **MkDocs:** `mkdocs.yml`, `docs/`.
- **VitePress / Nextra:** `.vitepress/config.ts` or `theme.config.js`.
- **Sphinx:** `conf.py`, `index.rst`, `_build/`.

Note where the docs site builds in `YORIENT.md` so contributors do not edit generated HTML.

## Obsidian / vault

**Manifest:** `.obsidian/` directory; no `package.json` equivalent.
**Plugins worth noting:** Dataview, Templater, QuickAdd, Periodic Notes, MOCs.
**Layout signal:** PARA folders (`01 Projects`, `02 Areas`, `03 Resources`, `04 Archive`), or `00 - Maps of Content/` for MOC-first vaults.
**Read first:** the top-level MOC or `INDEX.md` if one exists; otherwise the highest-level folder names.
**Commands live in:** there are none — Obsidian is read/write of files. Skip the `## Commands` section in `YORIENT.md` for vaults.

## Generic / unknown

If no manifest files match:

1. Check `README.md` for build instructions.
2. Check `Makefile`, `justfile`, `Taskfile.yml`, `shell.nix`, `flake.nix`.
3. Check `Dockerfile` for the assumed build environment.
4. List top-level files alphabetically and flag anything ending in `.toml`, `.yaml`, `.json`, or `.lock`.
5. Mark the ecosystem `unknown` in `YORIENT.md`; do not guess commands.
